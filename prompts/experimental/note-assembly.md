# 阶段 4：笔记组装

## 任务

将前三个阶段的产出组装成一份完整的 Markdown 笔记，可直接进入 Obsidian vault。

## 输入

你已经在当前对话中产出了以下内容：
1. 阶段 1：元数据 + 论文结构 + 证据片段池
2. 阶段 2：Figure 论证路径分析
3. 阶段 3：实验与参数
4. （如为交互模式）用户讨论内容

## 组装规则

### 模板

参考 `templates/note-template.md` 的骨架结构，将各阶段产出填入对应区块。

### 语言规则（最高优先级，严格执行）

1. **正文全部用中文撰写**，包括所有学术概念的翻译
2. **仅以下内容保留英文原文**：
   - 化合物名/分子名（如 NOD-NY, Zn-HN, TPEN, ZIF-8）
   - 仪器/方法缩写（如 HPLC, XRD, TEM, EPR, UV-vis, DAN）
   - 细胞系名称（如 A549, Raw264.7, HeLa）
   - 基因/蛋白专名（如 metallothionein, GFP）
3. **所有可翻译的学术概念必须用中文**，以下是常见示例：
   - photolysis → 光解
   - fluorescence → 荧光
   - probe → 探针
   - turn-on response → 荧光增强响应
   - absorption red-shift → 吸收红移
   - cation selectivity → 阳离子选择性
   - release → 释放
   - kinetics → 动力学
   - cytotoxicity → 细胞毒性
   - incubation → 孵育
   - uptake → 摄取
   - localization → 定位
   - retention time → 保留时间
   - linearity → 线性关系
   - correlation coefficient → 相关系数
4. **首次出现的重要概念**可在中文后括号标注英文（如"光诱导电子转移（PET）"），之后只用中文
5. **原文摘录保持英文不翻译**（blockquote 中的引用原文）
6. **数学表达式使用 LaTeX 语法**：行内用 `$...$`，独立公式用 `$$...$$`。下标用 `_{}`（如 `$k_{\text{obs}}$`、`$\Delta G$`），数学算子用 `\min`、`\log` 等标准命令。不要将公式写成纯文本

### 研究概述区块

- 不是机械复述 Abstract
- 用中文帮读者快速抓住主线：研究了什么、为什么做、做出了什么

### Figure 论证路径区块

- 直接使用阶段 2 的输出
- 确保格式与模板一致

### 实验与参数区块

- 直接使用阶段 3 的输出
- 确保参数名为中文、格式与模板一致

### My Understandings 区块

- **仅在交互模式下生成**
- 非交互模式：此区块完全不出现，不留空壳，不留占位符
- 交互模式内容来源：
  - **核心理解**: 提炼用户在讨论中表达的理解（用用户自己的语言）
  - **纠正记录**: 记录用户被纠正的认知（❌ 原以为 → ✅ 实际上）
  - **待解决的疑问**: 用户提出但未完全解答的问题
- 只记录实际讨论过的内容，跳过的 Figure 不补，没聊到的话题不编

### 生成信息

在笔记末尾添加一行：
```
> 由 paper-to-note skill 生成 | {当天日期 YYYY-MM-DD}
```

## 质量控制

在组装过程中严格执行以下检查：

1. **删除 AI 语气** — 不允许出现"好的"、"当然"、"让我来解释"、"接下来我们看看"等对话语气词
2. **不编造信息** — 不添加论文中不存在的数据、结论或推断
3. **对话不是事实来源** — AI 对话内容（包括交互模式）不能作为实验参数、作者 claim、figure 结论的证据，只能用于组织内容和记录用户思路
4. **信息不足标注** — 某个区块信息不足时标注 `[信息不足，建议查阅原文]`，不编造内容填充
5. **格式一致性** — 所有表格、引用格式必须统一
6. **中文检查** — 确认正文中没有出现未翻译的英文学术概念（化合物名和缩写除外）
7. **无行内锚点** — 最终笔记中不允许出现 `[§E...]`、`[E...]` 或任何形式的证据编号标记

## 文件名生成

格式：`{期刊缩写}--{年份}--{主要内容概括}.md`

规则：
- 期刊名空格用 `-` 替代（如 `ACS-Nano`）
- 主要内容用中文概括
- `--` 双连字符作分隔符
- 文件名不含空格
- 文件系统非法字符（`/ : * ? " < > |`）替换为 `-`

示例：`ACS-Nano--2024--高效MOF催化剂用于CO2还原.md`

## 输出

输出完整的 Markdown 笔记正文（不含代码块包裹），可直接写入 .md 文件。
