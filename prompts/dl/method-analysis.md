# 阶段 3：DL 方法分析

## 任务

基于阶段 1 提取的论文结构和关键原文片段，系统整理这篇深度学习论文的方法信息。重点不是“实验 1 / 实验 2”，而是把模型、训练配置、数据集、benchmark 和 ablation 组织成可回查的结构化笔记。

## 规则

### 1. 核心原则

**超参数必须绑定到它所属的上下文，绝不能混写。**

具体要求：
- 训练超参数只放在训练配置里
- 推理设置如果单独出现，要明确标注为推理，不要混入训练配置
- 模块内部参数要挂在对应模块描述下面
- benchmark 数字要和具体任务 / 数据集绑定，不能脱离上下文孤立罗列

### 2. Section 1：Model Architecture

先给出**总体架构**的一段话概括，再列出关键模块。

对每个关键模块，提取：
- 模块名
- 功能
- 结构细节（层、维度、激活、输入输出关系）
- 创新点（与标准做法的差异）

如果论文没有把模块拆得很细，也不要硬拆；按作者真实组织方式写。

### 3. Section 2：Training Configuration

使用下表格式整理训练配置：

| 超参数 | 值 |
|--------|-----|
| 优化器 | ... |
| 学习率 | ... |
| 学习率策略 | ... |
| 预热 | ... |
| 批大小 | ... |
| 训练轮数 | ... |
| 权重衰减 | ... |
| 数据增强 | ... |
| 正则化 | ... |
| 损失函数 | ... |
| 梯度裁剪 | ... |
| 混合精度 | ... |
| EMA 衰减率 | ... |
| 硬件 | ... |

补充规则：
- 参数名必须用中文
- 参数值保留原始数值、单位和英文专名
- 如果同一论文有多套训练设置，按任务或数据集拆开写，不能糊成一张表
- 在训练配置后附 1-2 段英文原文摘录

如果论文还给了明显独立的推理设置（如输入分辨率、beam size、NMS 阈值、test-time augmentation），可额外增加：

```markdown
### 推理设置（可选）

| 参数 | 值 |
|------|-----|
| ... | ... |
```

### 4. Section 3：Datasets

使用下表格式整理：

| 数据集 | 用途 | 规模 | 划分 |
|--------|------|------|------|
| ... | ... | ... | ... |

用途应说明它在论文里扮演什么角色，例如训练、验证、测试、零样本评估、下游迁移等。

### 5. Section 4：Benchmark Results

按**任务 / 数据集**组织 benchmark 结果，而不是把所有数字揉在一起。

每组 benchmark 至少包含：
- 任务或数据集标题
- 对比表格
- 1-2 段英文原文参考

表格格式：

| Method | Metric1 | Metric2 | ... |
|--------|---------|---------|-----|
| ... | ... | ... | ... |

要求：
- 明确标出本文方法，优先写成 `本文方法（模型名）` 或在表格中加粗
- 如果作者同时报告性能和复杂度（如 FLOPs / Params / FPS / Latency），可并入同一组 benchmark
- 不要省略单位和评价协议中的关键限定条件

### 6. Section 5：Ablation Studies

对每组 ablation 单独整理，使用下表格式：

| Config | Metric | Change |
|--------|--------|--------|
| ... | ... | ... |

每组表格后补一句中文结论，说明作者借此证明了什么。

如果论文没有独立 ablation，明确写 `[未提供独立消融实验]`。

### 7. 语言规则（最高优先级，严格执行）

1. 正文说明用中文撰写
2. 以下内容保留英文：
   - 模型名 / 架构名：`ResNet`, `ViT`, `YOLO`, `U-Net`, `DETR`, `SAM`, `GAN`, `Diffusion`
   - 指标缩写：`mAP`, `IoU`, `FID`, `PSNR`, `SSIM`, `BLEU`, `Top-1`, `AUC`
   - 数据集名：`ImageNet`, `COCO`, `VOC`, `ADE20K`, `Cityscapes`, `CIFAR-10`
   - 层 / 模块名：`Conv`, `BN`, `LN`, `ReLU`, `GELU`, `Softmax`, `MLP`, `Patch Embedding`
   - 优化器名 / 损失函数名在作为专有值时可保留英文，如 `AdamW`, `SGD`, `Cross-Entropy`
3. 以下常见术语统一译成中文：
   - `feature map` → 特征图
   - `backbone` → 骨干网络
   - `attention` → 注意力
   - `pooling` → 池化
   - `skip connection` → 跳跃连接
   - `head` → 检测头 / 任务头（按上下文）
   - `neck` → 特征融合层
   - `anchor` → 锚框
   - `embedding` → 嵌入
   - `fine-tuning` → 微调
   - `pre-training` → 预训练
   - `inference` → 推理
   - `epoch` → 训练轮数
   - `dropout` → 随机失活
   - `batch normalization` → 批归一化
   - `layer normalization` → 层归一化
   - `encoder` → 编码器
   - `decoder` → 解码器
   - `upsampling` → 上采样
   - `downsampling` → 下采样
   - `bounding box` → 边界框
   - `ground truth` → 真值标注
   - `feature pyramid` → 特征金字塔
   - `depthwise separable convolution` → 深度可分离卷积
   - `data augmentation` → 数据增强
   - `loss function` → 损失函数
   - `weight decay` → 权重衰减
4. 首次出现的重要概念可写成”中文（English）”，之后优先使用中文
5. 原文摘录始终保持英文，不翻译
6. 数学表达式使用 LaTeX 语法：行内用 `$...$`，独立公式用 `$$...$$`。下标用 `_{}`（如 `$p_{\text{data}}$`），数学算子用 `\min`、`\max`、`\log`，期望用 `\mathbb{E}`。不要将公式写成纯文本（如 `p_data`、`E[...]`）
